---
name: MEQ ID numbering rule
description: How to assign IDs when adding new MEQ cases from ChatGPT/Claude output
---

When the user uploads a new batch of MEQ cases generated by ChatGPT or Claude, those files will contain their own ID numbers (e.g. meq_018, meq_023, meq_005 — whatever the AI chose).

**Always ignore those IDs.**

Instead:
1. Check the current highest meq_NNN in the MEQ_BANK (grep for `id: .*meq_` in DailyMEQMode.tsx)
2. Assign the next batch starting from that number + 1
3. Example: if bank currently ends at meq_026, next batch is meq_027, meq_028, meq_029…

**Why:** ChatGPT/Claude do not have live access to the bank and will reuse or conflict with existing IDs. Continuous sequential numbering from the current last entry is the only safe rule.

**Also apply to question-registry.json:** use the same renumbered IDs there too.
